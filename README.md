## Лабораторные работы №1–6 (кратко)

---

### Лабораторная работа №1. Деплой на Render и шаблонизация

**Что сделали:** Перенесли готовый фронтенд в NestJS, настроили шаблонизатор EJS, выделили повторяющиеся блоки (header, footer, nav, session), задеплоили на Render.

**Результат:** Сайт работает в интернете, есть два состояния сессии (авторизован/неавторизован).

---

### Лабораторная работа №2. Создание доменной модели

**Что сделали:** Спроектировали базу данных для трекера привычек (6 таблиц: User, Habit, Category, HabitLog, Achievement, UserAchievement), создали ER-диаграмму.

**Результат:** Готовая схема БД, развёрнутая на Render.

---

### Лабораторная работа №3. MVC + База данных + SSE

**Что сделали:** Подключили базу к сайту, данные сохраняются и читаются из PostgreSQL. Сделали CRUD (создание, чтение, обновление, удаление) привычек. Добавили уведомления в реальном времени (SSE) при создании привычки.

**Результат:** Полноценный сайт с живыми данными и уведомлениями.

---

### Лабораторная работа №4. REST API

**Что сделали:** Создали отдельный API для программ на `/api/habits`. Добавили пагинацию (постраничный вывод), валидацию входных данных, Swagger документацию и единый фильтр ошибок.

**Результат:** JSON API, который могут использовать мобильные приложения и другие сервисы.

---

### Лабораторная работа №5. GraphQL API

**Что сделали:** Сделали ещё один API — GraphQL на `/graphql`. Клиент сам выбирает, какие поля ему нужны. Добавили ограничение сложности запросов (max 500).

**Результат:** Гибкий API, где одним запросом можно получить привычку и связанного пользователя.

---

### Лабораторная работа №6. BFF (Interceptor + Кэш + Загрузка файлов)

**Что сделали:**
- **Interceptor** — замеряем время ответа, выводим в консоль и в заголовок `X-Response-Time`.
- **Кэш** — GET `/api/habits` кэшируется на 10 секунд, повторные запросы мгновенные.
- **Загрузка файлов** — картинки загружаются в Yandex Cloud, возвращается публичная ссылка.

**Результат:** Сервер быстрее (кэш), умеет загружать файлы в облако, показывает время ответа.

---

## Итог: что умеет проект

| Функция | Как проверить |
|---------|---------------|
| Сайт | `http://localhost:3000/habits` |
| REST API | `GET /api/habits` |
| GraphQL | `POST /graphql` |
| Уведомления | Создать привычку → уведомление на другой вкладке |
| Кэш | Два одинаковых запроса → второй за 0ms |
| Загрузка файлов | `POST /upload/habit-image` |
| Документация | `GET /api-docs` (Swagger) |
